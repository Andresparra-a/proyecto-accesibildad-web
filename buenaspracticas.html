Guía de Buenas Prácticas Esenciales (WCAG)

Aplicar estas prácticas garantiza que su contenido sea utilizable y comprensible para una audiencia más amplia, incluyendo personas que dependen de tecnologías de asistencia.

  • Texto Alternativo

    Todas las imágenes que transmitan información deben tener un atributo alt descriptivo. Si una imagen es puramente decorativa, el alt debe dejarse vacío (alt="") para que los lectores de pantalla la ignoren.

    <img src="foto.jpg" alt="Un desarrollador escribiendo código en una laptop.">

  • Contraste de Color Suficiente

    El color del texto y el color de fondo deben tener una relación de contraste mínima de **4.5:1** (Nivel AA de WCAG) para asegurar que las personas con baja visión puedan leer el contenido sin esfuerzo.

  • Navegación por Teclado

    Todos los elementos interactivos (enlaces, botones, campos de formulario) deben ser accesibles y operables **utilizando únicamente la tecla TAB**. El foco visible (el borde que aparece al presionar TAB) debe ser claro.

  • Uso Correcto de Etiquetas y Roles

    Utiliza las etiquetas semánticas adecuadas. Por ejemplo, usa la etiqueta **<button>** para acciones y la etiqueta **<a>** para enlaces, en lugar de usar un simple <div>. Esto da el "rol" correcto a los elementos.

  • Etiquetas de Formulario (label)

    Cada campo de entrada en un formulario (<input>, <textarea>) debe estar asociado con una etiqueta <label> utilizando el atributo for. Esto permite a los lectores de pantalla anunciar qué información se espera en ese campo.

    Ejemplo: <label for="nombre">Nombre:</label>
