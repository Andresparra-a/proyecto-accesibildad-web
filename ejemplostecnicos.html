Errores comunes y soluciones de accesibilidad

Ahora se mostrará una tabla comparativa de codificación que normalmente rompen la accesibilidad y de qué forma se puede corregir la semántica nativa de HTML5

Tabla de correciones para la accesibilidad web
Problema Solución recomendada Concepto clave
Usar<div> para un botón. Usar la etiqueta semántica <button>. *semántica nativa*.El lector de la pantalla lo identifica como un botón automáticamente
Ícono sin texto descriptivo Añadir aria-label="Cerrar ventana" al ícono **aria-label:**Esto proporciona una etiqueta invisible para los usuarios videntes pero es visible para las tecnologías de asistencia
Un campo de entrada de datos que no poseen etiqueta <label<. Asociar al campo con un<label for="..."<. **Etiquetas:** Le permite al usuario poder hacer click en la etiqueta para enfocar el campo y le comunica al lectorel propósito del campo
Un elemento no puede cambiar de color al pasar el ratón. Asegurar que el estado del hover y el foco sean claramente visibles. **Foco visible:** Es necesario para la navegación por teclado
